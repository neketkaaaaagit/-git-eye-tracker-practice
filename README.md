# Eye Tracking Research 

Веб-стенд для исследования методов и технологий трекинга взгляда. Проект ориентирован на учебную и исследовательскую практику по теме мониторинга визуальной работы специалиста, в том числе рентгенолога.

## Что уже реализовано

* трекинг взгляда через веб-камеру в браузере;
* 9-точечная калибровка;
* тестовая рабочая область с изображением;
* heatmap распределения внимания;
* разбиение изображения на AOI-зоны 3×3;
* счётчики фиксаций и морганий;
* экспорт результатов сессии в CSV и JSON;
* запуск локально, через Docker и публикация через GitHub Pages;
* CI/CD workflow для smoke-проверок, сборки и деплоя.

## Почему проект подходит под практику

Проект можно оформить как **контейнеризированный исследовательский стенд** для анализа стратегий просмотра изображений. Это закрывает основные требования практики:

1. **Проектирование развёртывания** — описан стек, архитектура и сценарии запуска.
2. **Процесс тестирования** — есть ручной чек-лист и автоматический smoke-тест.
3. **Оформление результата в виде Docker-образа** — добавлены `Dockerfile`, `compose.yml`, `nginx/default.conf`.
4. **Поставка на конвейер CI/CD** — добавлены workflow для CI и GitHub Pages.
5. **Итоговый отчёт** — в `docs/` лежат шаблоны и материалы для пояснительной части.

## Архитектура

* **Frontend:** HTML + CSS + JavaScript
* **Computer vision:** MediaPipe Face Landmarker for Web
* **Контейнеризация:** Nginx в Docker
* **CI:** GitHub Actions

## Быстрый запуск

### Вариант 1. Локально через Node.js

После установки Node.js:

```bash
npm run dev
```

Открой в браузере:

```text
http://localhost:8080
```

### Вариант 2. Локально через Python

```bash
python -m http.server 8080
```

Открой:

```text
http://localhost:8080
```

### Вариант 3. Через Docker

Сборка образа:

```bash
docker build -t eye-tracker-stand .
```

Запуск контейнера:

```bash
docker run --rm -p 8080:80 eye-tracker-stand
```

Открой:

```text
http://localhost:8080
```

### Вариант 4. Через Docker Compose

```bash
docker compose up --build
```

## Как использовать стенд

1. Открой приложение по `localhost` или через HTTPS.
2. Нажми **«Включить камеру»**.
3. Разреши браузеру доступ к камере.
4. Выполни **калибровку 9 точек**.
5. При необходимости загрузи тестовое изображение.
6. Нажми **«Начать сессию»**.
7. Выполни наблюдение, после чего экспортируй CSV/JSON.

## Ограничения

* это не медицинская система и не сертифицированный eye tracker;
* точность зависит от освещения, положения головы и качества камеры;
* для корректной работы камеры нужна безопасная среда: `localhost` или HTTPS;
* MediaPipe и модель загружаются из CDN и облачного хранилища, поэтому нужен интернет.

## Рекомендация под тему рентгенолога

Для практики используй обезличенные тестовые изображения и сравнивай:

* траектории взгляда;
* долю времени по AOI;
* частоту фиксаций;
* влияние разных вариантов разметки или интерфейса просмотра.
