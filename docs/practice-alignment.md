# Соответствие этапам практики

## Тема
**Исследование методов и технологий трекинга взгляда для мониторинга работы рентгенолога**

## Как проект соответствует плану практики

### 1. Инструктаж обучающегося
Формальный этап. Отражается в календарном плане и не требует отдельной функции в приложении.

### 2. Проектирование развёртывания практического решения
В проекте предусмотрены:

- архитектура стенда: браузерный frontend + MediaPipe;
- локальный запуск, контейнерный запуск и публикация на GitHub Pages;
- конфигурация Nginx для статического стенда;
- документация по запуску и деплою.

### 3. Процесс тестирования
Организованы два уровня проверки:

- **автоматический smoke-тест** (`npm test`) — проверяет наличие ключевых файлов и критичных частей интерфейса;
- **ручные сценарии** — описаны в `docs/test-plan.md` и покрывают запуск камеры, калибровку, heatmap, экспорт и работу контейнера.

### 4. Оформление результата в виде Docker-образов или конфигураций
Результат оформлен через:

- `Dockerfile`;
- `compose.yml`;
- `nginx/default.conf`.

Это позволяет воспроизводимо развернуть стенд на любой машине с Docker.

### 5. Поставка на конвейер CI/CD
В репозитории предусмотрены workflow:

- `.github/workflows/ci.yml` — smoke-test, build, docker build;
- `.github/workflows/pages.yml` — публикация статической версии на GitHub Pages.

### 6. Итоговый отчёт
В `docs/` подготовлены материалы для отчёта:

- `deployment.md`;
- `test-plan.md`;
- `report-outline.md`.

## Научно-практическая связка с темой рентгенолога
Проект можно описывать как **исследовательский тестовый стенд для анализа визуального поведения при изучении диагностических изображений**. Для этого в приложении уже есть:

- загрузка тестового изображения;
- тепловая карта внимания;
- AOI-анализ по зонам;
- фиксация морганий и фиксаций;
- экспорт данных для отчёта или внешнего анализа.

## Что писать в выводе
Разработан и подготовлен к развёртыванию веб-стенд, обеспечивающий сбор и визуализацию данных о предполагаемом направлении взгляда пользователя на тестовом изображении. Стенд контейнеризирован, сопровождается документацией, тестовым контуром и CI/CD-конвейером, что соответствует требованиям практической части.
