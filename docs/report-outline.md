# Каркас итогового отчёта

## 1. Введение
- актуальность трекинга взгляда;
- связь с мониторингом работы рентгенолога;
- цель практической части.

## 2. Постановка задачи
- разработать развёртываемое практическое решение;
- обеспечить контейнеризацию;
- организовать тестирование;
- подготовить CI/CD.

## 3. Выбор технологий
- HTML/CSS/JavaScript;
- MediaPipe Face Landmarker;
- Docker + Nginx;
- GitHub Actions;
- GitHub Pages.

## 4. Архитектура решения
- структура проекта;
- сценарий получения данных с камеры;
- калибровка;
- AOI-анализ;
- heatmap;
- экспорт результатов.

## 5. Развёртывание
- локальный запуск;
- контейнерный запуск;
- публикация через GitHub Pages.

## 6. Тестирование
- автоматический smoke-test;
- ручные сценарии;
- результаты проверки.

## 7. Практический результат
Опиши, что разработан исследовательский стенд для регистрации и визуализации предполагаемого направления взгляда пользователя на диагностическом изображении. Укажи, что стенд позволяет:

- провести калибровку;
- собрать данные сессии;
- построить heatmap;
- оценить время по AOI-зонам;
- экспортировать результаты.

## 8. Скриншоты
Добавь:

- стартовую страницу;
- активную камеру;
- процесс калибровки;
- heatmap после сессии;
- таблицу AOI;
- страницу GitHub Actions или адрес размещения.

## 9. Заключение
Сформулируй, что требования практики выполнены: разработано практическое решение, оформлена контейнеризация, настроен CI/CD, подготовлены тестовые сценарии и документация.
