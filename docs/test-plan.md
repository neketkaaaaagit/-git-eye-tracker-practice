# План тестирования

## 1. Цель тестирования
Подтвердить, что исследовательский стенд:

- запускается локально и в контейнере;
- корректно запрашивает доступ к камере;
- позволяет выполнить калибровку;
- собирает данные сессии;
- отображает heatmap и AOI-метрики;
- экспортирует результаты.

## 2. Автоматические проверки

### Smoke-тест
Команда:

```bash
npm test
```

Проверяет:

- наличие основных исходных файлов;
- наличие workflow CI/CD;
- наличие Docker-конфигурации;
- наличие ключевых UI-блоков и функций приложения.

## 3. Ручные сценарии

### Сценарий 1. Запуск локально
1. Выполнить `npm run dev` или `python -m http.server 8080`.
2. Открыть `http://localhost:8080`.
3. Убедиться, что страница загружается без ошибок.

**Ожидаемый результат:** интерфейс приложения отображается полностью.

### Сценарий 2. Доступ к камере
1. Нажать **«Включить камеру»**.
2. Разрешить доступ к камере.

**Ожидаемый результат:** отображается видеопоток и разметка глаз/радужки.

### Сценарий 3. Калибровка
1. Нажать **«Калибровка 9 точек»**.
2. Поочерёдно сохранять 9 точек.

**Ожидаемый результат:** статус калибровки меняется на «Готово», точка взгляда становится стабильнее.

### Сценарий 4. Сбор данных сессии
1. Нажать **«Начать сессию»**.
2. Перемещать взгляд по разным зонам изображения 20–30 секунд.
3. Нажать **«Остановить сессию»**.

**Ожидаемый результат:** увеличиваются счётчики точек, времени, зон и фиксаций.

### Сценарий 5. Heatmap и AOI
1. Запустить сессию.
2. Сконцентрировать внимание на нескольких разных участках изображения.

**Ожидаемый результат:** на рабочей области появляется heatmap, таблица AOI показывает распределение времени.

### Сценарий 6. Экспорт данных
1. После сессии нажать **«Экспорт CSV»**.
2. Затем нажать **«Экспорт JSON»**.

**Ожидаемый результат:** скачиваются два файла с данными наблюдения.

### Сценарий 7. Docker-запуск
1. Выполнить `docker compose up --build`.
2. Открыть `http://localhost:8080`.

**Ожидаемый результат:** приложение доступно из контейнера.

## 4. Критерии успешности

Стенд считается готовым к демонстрации и включению в отчёт, если:

- все сценарии 1–7 проходят без критичных ошибок;
- экспорт создаёт непустые файлы;
- Docker-сборка выполняется успешно;
- CI workflow проходит без ошибок.
